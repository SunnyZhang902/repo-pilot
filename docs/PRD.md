# RepoPilot 产品需求文档（PRD）

## 1. 项目背景（Background）

开发者在接手一个新的代码仓库时，通常需要花费大量时间阅读 README、分析目录结构、搜索关键函数以及梳理模块之间的调用关系。

对于中大型项目，这一过程往往需要数小时甚至数天，学习成本较高，也影响了开发效率。

随着大语言模型的发展，AI 已具备较强的代码理解能力，但目前多数 AI 编程工具更关注**代码生成**，而不是**帮助开发者快速理解已有代码**。

因此，我们希望构建一个面向开发者的 AI Repository Understanding Platform，让开发者能够通过自然语言快速理解任何代码仓库。

---

## 2. 产品目标（Vision）

帮助开发者在几分钟内理解一个陌生项目，而不是花费几个小时阅读代码。

一句话定位：

> **Understand any codebase in minutes with AI.**

---

## 3. 目标用户（Target Users）

### 用户一

刚加入公司的开发者

需要快速熟悉项目。

---

### 用户二

准备参与开源贡献的开发者

需要理解项目结构。

---

### 用户三

学生

需要阅读课程项目或实验项目。

---

## 4. 用户痛点（Pain Points）

目前开发者阅读陌生项目主要存在以下问题：

* README 信息不足
* 项目目录复杂
* 调用链难以追踪
* 模块关系不清晰
* 很难快速定位核心代码
* 新人学习成本高

---

## 5. MVP（第一阶段）

第一版仅实现以下能力：

### Repository Import

输入 GitHub Repository。

---

### Repository Parsing

自动解析项目结构。

---

### Repository Chat

支持自然语言问答。

例如：

* 登录逻辑在哪里？
* 数据库在哪里？
* 核心模块有哪些？

---

### Architecture Diagram

自动生成项目架构图。

---

### Documentation Generation

自动生成项目文档。

---

## 6. 第一阶段明确不做（Non-goals）

为了保证项目能够快速完成，第一版**不实现**以下功能：

* 自动修改代码
* 自动提交 Git
* 自动生成 Pull Request
* 多 Agent 协同
* 自动修复 Bug

这些能力将在后续版本中考虑。

---

## 7. 成功标准（Success Metrics）

RepoPilot 第一版完成后，应满足以下目标：

* 能够导入 GitHub Repository
* 能够解析项目结构
* 能够回答代码相关问题
* 能够引用具体文件位置
* 能够生成基础架构图
