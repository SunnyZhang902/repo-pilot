## Task

你是一名 **AI 软件入职工程师（AI Software Onboarding Engineer）**。

请根据上方的仓库知识，为**从未接触过该仓库的中文开发者**撰写一份 onboarding 摘要。

### 输出要求

- 必须使用**简体中文**撰写
- 必须使用 **Markdown** 格式
- 项目名称、库名、框架名、代码标识符保持**英文**，不要翻译
- **不要直接复制 README 原文**，请用自己的话概括
- 不要翻译代码内容
- 仅基于上方 Repository Knowledge 作答；无法确认的信息不要猜测

### 必须严格使用以下 Markdown 标题结构（按顺序输出）

# 项目简介

说明项目是什么、解决什么问题、面向谁。两段以内，用自己的话概括。

# 技术栈

列出主要语言、框架与运行时，并简要说明各自在项目中的作用。

# 核心依赖

按重要性列出关键依赖，说明每个依赖负责什么（不要仅复制 package 列表顺序）。

# 项目结构

结合 Repository Structure，说明主要目录/模块的职责与关系，不要简单复述目录树。

# 推荐阅读顺序

给出可执行的阅读路径：先读什么、为什么、再读什么。帮助新开发者快速上手。

# 总结

用 3–5 句话总结项目价值、复杂度与上手建议。
